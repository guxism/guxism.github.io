A Misconcept

Overloading appearls to be a more obscure concept in C++. Previously, I thought that overloading means different function signatures with a same name, by signature, I mean things of form return_type(argument_list), which can be used to define function pointer:

typedef void(*f)(int)

But according to The signature of a function, signature must include function name. So my understanding was wrong!

Read Function Overloading in cppreference.

it says:

Functions are identified not only by their names, but also by their arguments.

No words about return type.

How do you call return_type(argument_list)

By chance I enter the realm of std::function. Note that cppreference use different notation: R(Args...)

function is a class template, with specified R(Args..), a class is generated by compiler.

But it does not mention what the f* is R(Args...), but it does name R as result_type, if Args... has only 1 argument, then argument_type is the type of the argument.

The answer is: you can't call it by a name.

Why should we not care about reture_type

double f(int a, int b) {}
int f(int a, int b) { return 1;}

CPPLINT(whatever) will tell you : can't overload functions distinguished by return type alone.

I think I know why. Suppose the above two lines can compile, in the following code, which f is called?

f(1, 2);

the parser is confused.

It is like you can't have operator** for pow() like that in python, for if there is a pointer p, how does the parser parse this line: some_object ** p ?
