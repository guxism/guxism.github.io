Blockchain's First Soft Touch

Year 2022, I set out to learn some blockchain basics, I have just roughly finished the book mastering bitcoin but didn't understand a thing about it, I guess I need to code to understand, so here I go.

First open Remix, a online IDE for deploying and running transactions, and paste the code from this nicely written tutorial by Microsoft, with minor modification like this as suggested by Dr. G:

// SPDX-License-Identifier: GPL-3.0
pragma solidity >0.7.0 <0.8.0;

contract Marketplace {
    address public seller;
    address public buyer;
    mapping (address => uint) public balances;

    event ListItem(address seller, uint price);
    event PurchasedItem(address seller, address buyer, uint price);

    enum StateType {
          ItemAvailable,
          ItemPurchased
    }

    StateType public State;

    constructor() public {
        seller = msg.sender;
        State = StateType.ItemAvailable;
    }


    function initialBalance(address participant, uint amount) public {
        require(msg.sender == participant, "you cannot update someone else's balance");
        balances[participant] = amount;
    }


    function buy(address seller, address buyer, uint price) public payable {
        require(price <= balances[buyer], "Insufficient balance");
        State = StateType.ItemPurchased;
        balances[buyer] -= price;
        balances[seller] += price;

        emit PurchasedItem(seller, buyer, msg.value);
    }
}

Set the environment to Remix VM(London), something that Solidity programs run on. You can set up your own environment. But now we just want to try it out.

Then choose an account from the drop-down list of ACCOUNT, for example, 0x0A098Eda01Ce92ff4A4CCb7A4fFFb5A43EBC70DC, copy that and paste in your notebook. then click the yellow Deploy button to deploy a transaction. Note that transaction is not smart contract, it is the place where smart contracts interact with each other, transactions are like multi-universes, if I initialize A in another transaction, in this transaction, nothing is affected.

Make sure we go to the same marketplace by pasting the marketplace address at the At Address inputbox:

Choose 0x4B0897b0513fdC7C541B6d9D7E929C4e5364D2dB as B. Initialize B's balance. After that, we let A buy stuff from B at the price of 20 ETH, we can see that in this marketplace, B gains 20 ETH, now it has 120 ETH, and A has 80 ETH left.


The "State" shows that the Item is purchased:

    enum StateType {
          ItemAvailable,
          ItemPurchased
    }

According to the blockchain theory, there is one thing left to do, that is to push the transaction to the blockchain. I have no idea how to do that on the London VM, it seems to have forbidden me from publish my contract and advices me to build a local blockchain myself.

Then I follow the rest of the tutorial and find that this hardly interests me, because it is opaque, one can hardly see what happens behind the scene, there are tools and frameworks that together construct an inaccessible labyrinth, the amount of information is overwhelmingly huge. You can take a look at my summary of reading. I tumbled upon Truffle, OpenZeppelin, Ganache, Testnets for one entire day and knew how to interact with Infuro, but I felt rather frustrated in the end.

Of course, this is not the end of my journey of blockchain. A lot of articles and tutorials await, I decide to take at least one hour a day to read about the subject. Hope that will change my mindset some time.
