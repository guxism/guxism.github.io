RSA 的数学推理

写这篇文章之前，我看了三本书相关内容：

  1. 图解密码技术 [日]结城浩，没有给出数学证明
  2. Computer Networking，James F. Kurose, Keith W. Ross，有部分数学推理，但关键部分只引用第三本书的证明
  3. NETWORK SECURITY PRIVATE Communication in a PUBLIC World，CHARLIE KAUFMAN • RADIA PERLMAN • MIKE SPECINER，看了这本书我才理解其中的数学原理，但这本书也没提供严格的数学证明，不过我认为已经够用了，我们又不是研究数学的人。

模运算

x mod n+y mod n=(x+y) mod n
x mod n×y mod n=(x×y) mod n

一开始我就遇到一个严重的问题，我不理解为什么以上两个等式成立。但如果我接受它们，我就能推出以下两个等式：

(x mod n)2=(x mod n)×(x mod n)=(x×x) mod n
(x mod n)y=xy mod n

Multiplicative inverse

通过观察发现存在两个质数 p, q, 使得

p mod n×q mod n=1

这是另外一个无法证明的地方，不过也可以通过举例理解，假设 n=10,p=3,q=7

Totient function

将与 n 互质(relatively prime)的数的总量记为 ϕ(n)，包括如果 n 是质数，那么 ϕ(n)=n−1 。现在这个性质是何用处未知，我的理解这是数学家探索质数的性质的时候偶尔发现的性质。

如果 n 是两个质数 p, q 的乘积，那么 ϕ(n)=(p−1)×(q−1)，因为对于所有 x, 使得 p×x<n 的 x 不超过 q 个，包括 0 在内，即 x∈[0,q), 一共是 q 个数，同理，对于所有 y, 使得 q×y<n 的 y 的个数一共有 p 个，其中也包括 0, 所以从 0 到 pq 一共 pq 个小于 pq 的数中，有 p+q−1 个数是不符合“与 pq 互质”这一条件的，减去 1 是因为 0 被包含两次。

所以对于质数 p, q, ϕ(pq)=pq−(p+q−1)=(p−1)(q−1)

再次说明，我不理解为什么要推出这种结论，我的理解是数学家随手造出一个结论，然后把结论放在工具箱里面，说不定哪天就可以用。

Modular Exponentiation

通过观察发现以下性质可能成立：

xy mod n=x(y modϕ(n)) mod n

对于某些数是不能成立的，比如 n=10,x=4,y=8,ϕ(n)=4 的时候，等号左边等于 6，右边等于 1。

>>> (4 ** 8) % 10
6
>>> (4 ** (8 % 4)) % 10
1

但如果 x, y 都是质数，而且 n 是两个质数的乘积，那么上面的等式成立。

同样，我无法理解它为什么能成立，因为偶然性太大了，你怎么能偶然发现 ϕ(n), 同时又恰好发现可以 y mod ϕ(n)?

加解密的机制

对称加解密的设计是对于明文 m, 密钥 K, 通过 m XOR K = c 加密，通过 c XOR K 解密。同样，无法理解为什么这么刚好这么巧。

而不对称加解密则是，对于明文 m, 用公钥 k+ 加密: c=E(K+,m)，用 k− 解密: m=D(K−,c)

通过模运算实现不对称加解密

假设某一个天，数学家或密码学家观察了这个等式很久

xy mod n=x(y modϕ(n)) mod n

发现如果 y modϕ(n) 如果等于 1， 那么直接就有 xy mod n=x mod n 。

如果 x=med ，m 是密文，me 是加密之后的密文，(me)d 是解密，那么可以看出 (me)d=med, 如果能让 med=m 那就实现了解密。

最后一次说明，这里偶然性太多，我也没看过数学家的传记，所以我同样无法理解怎么必然得出这些结论。

决定论认为，存在一个拉普拉斯恶魔，只要知道 x mod n×y mod n=(x×y) mod n, 必然能推出 RSA。不幸的是我们不是拉普拉斯恶魔。

所以密码学家利用了 xy mod n=x(y modϕ(n)) mod n 并利用 y modϕ(n)=1 这一特例，设计出 RSA。

RSA 的设计

首先生成两个质数，p, q, 得出 n=pq, 以及 z=ϕ(n)=(p−1)(q−1)

通过计算得出 e, e<n ，并且 e 也是一个质数，因为这一才能满足 xy mod n=x(y modϕ(n)) mod n

通过计算得出一个质数 d, 使得 ed mod z=1, 这样才能满足 y modϕ(n)=1

将 <e,n> 作为公钥，将 <d,n> 作为私钥。

加密的时候，对于明文 m, 进行以下运算：

c=me mod n
解密的时候，对于密文 c, 进行以下运算：

cd mod n=(me mod n)d=med mod n=med modz mod n=m mod n

由于设计了 m<n, 所以 m mod n=m
