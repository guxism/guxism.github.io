分区

最近 Linux 系统上磁盘不够用, 于是在双系统中的 Windows 中压缩出 30G, 还有为保护 SSD 所以未分配的 40G, 我的目标是将它们投入使用

挂载

我们需要让分区自动挂载. 首先将分区挂在任意目录, 找到分区的 UUID:

╭─/dev/disk/by-uuid  
╰─➤  ls -alh
total 0
lrwxrwxrwx 1 root root  15 10月 31 17:09 c5afdb70-b9d1-487f-b219-970e2e693e2a -> ../../nvme0n1p5

在 /etc/fstab 写入:

UUID=c5afdb70-b9d1-487f-b219-970e2e693e2a /nvme0n1p5      ext4    errors=remount-ro 0       0

可参考我之前写的文章

创建挂载目录

╭─/  
╰─➤  mkdir /nvme0n1p5

刷新:

╰─➤  mount -a
╰─➤  df -h | grep nvme0n1p5 
/dev/nvme0n1p5   32G   49M   30G   1% /nvme0n1p5

更新权限

➜  / sudo chown root_:root_ /nvme0n1p5 -R

没找到将多个分区挂在一个目录的方法. 目前也不考虑将新分区和 Linux 主分区合并, 因为新分区在主分区前面, 如果合并可能会导致文件拷贝.

所以我目前的做法是用软链接欺骗桌面系统, 腾出空间给其他应用

Downloads -> Downloads2
Downloads2 -> /nvme0n1p5
