到 substrate 代码 frame/support/procedural/src/pallet/parse/call.rs:68~73

/// Attributes for functions in call impl block.

看看 pallet 代码做了什么：

#[pallet::weight(195_000_000)]

substrate 如何解析这串代码：

  1. 去掉 #
  2. 找关键字 weight
  3. 找到 195_000_000

我关心的是 #[pallet::call_index(expr)], 但我们写的 pallet 并不使用这一标志，那么我猜想也许 #[pallet::call] 做了 call_index 的事

看文档：

  • https://docs.substrate.io/reference/frame-macros/
  • https://paritytech.github.io/substrate/master/frame_support/attr.pallet.html#call-palletcall-optional
  • https://paritytech.github.io/substrate/master/frame_support/pallet_macros/attr.call_index.html

分析：

  1. 首先 #[pallet::call] 是 implementation of pallet dispatchables
  2. dispatchable 是指可分配/调度，是 dispatcher 的目标
  3. 每一个 dispatchable 必须定义 weight, 这个我们不关心
  4. 每一个 dispatchable 可以定义 call_index: 如果定义了，则 explicitly 定义 codec index in the Call enum, explicitly 暗示了 inexplicitly, 默认从 0 开始

OK, 我就接受了这种设定：transfer 这一 pallet dispatchable 位于 index 5200, 前端会从后端拉取一个 dispatchables 列表，这是必须的，但是怎么实现我们也不关心。

pub struct MultiLocation {

还有一大段注释我就不贴了，大意是这是一个 relative path between state-bearing consensus systems, 我觉得最后几个单词可以去掉，就叫 relative path. 注意到为什么叫做 Location => 联想到 URL 的 L

可以看到 parents 的注释：在这个 L(URL/Location) 前面的 parent junctions, 根据我们的经验，junction 就是 Enum 的一个 variant, 也就是一个参数，我之前称它为定语.

transfer 函数中，第一个参数，asset 的 parents 是 0，第二个参数 address 的 parents 是 1, coincidence? don't think so

看 Junction 定义：

/// Non-parent junctions that can be constructed, up to the length of 8. This specific `Junctions`
/// implementation uses a Rust `enum` in order to make pattern matching easier.
///
/// Parent junctions cannot be constructed with this type. Refer to `MultiLocation` for
/// instructions on constructing parent junctions.
#[derive(Clone, Eq, PartialEq, Ord, PartialOrd, Encode, Decode, Debug, TypeInfo, MaxEncodedLen)]
pub enum Junctions {
	/// The interpreting consensus system.
	Here,
	/// A relative path comprising 1 junction.
	X1(Junction),
	/// A relative path comprising 2 junctions.
	X2(Junction, Junction),
	/// A relative path comprising 3 junctions.
	X3(Junction, Junction, Junction),
	/// A relative path comprising 4 junctions.
	X4(Junction, Junction, Junction, Junction),
	/// A relative path comprising 5 junctions.
	X5(Junction, Junction, Junction, Junction, Junction),
	/// A relative path comprising 6 junctions.
	X6(Junction, Junction, Junction, Junction, Junction, Junction),
	/// A relative path comprising 7 junctions.
	X7(Junction, Junction, Junction, Junction, Junction, Junction, Junction),
	/// A relative path comprising 8 junctions.
	X8(Junction, Junction, Junction, Junction, Junction, Junction, Junction, Junction),
}

X2 就是我们选的 dest, 但 parents = 1，意味着 MultiLocation 是最后一个 Junction, 而 Junctions::Here 则表示不需要任何一个定语：那么就是 local network? 或者 local network 的自己，因为还有一种可能: local network, peer

仅凭注释是看不出什么问题，而代码如果跑不动也没有意义：最后找到一篇波卡的官方文档：https://polkadot.network/blog/xcm-the-cross-consensus-message-format/

https://polkadot.network/blog/xcm-the-cross-consensus-message-format/#-locations-in-xcm 解释了 XCM 的 Location, 我用 LInux 的文件系统来做个比喻

  1. Here 表示 .，表示当前路径, 在波卡生态中表示当前的链
  2. 在当前的 parachain 中， Here 表示当前的 parachain，parents 表示 .., 对应 LInux 文件系统的上一目录，parents 等于几就相当于有几个 ..
  3. 在当前的 Parachain 中，parents = 1 表示相对路径从 ../开始
  4. parents = 1, interiors = { parachain(2023), address: 0x123.. } 表示 ../parachain(2023)/0x123...4
  5. 反过来，代码注释中：The number of parent junctions at the beginning of this MultiLocation. 的 parent junctions 原来就是指 .., 如果不经代码作者（相关人员）解释，我觉得无法从这句话中得出什么。
