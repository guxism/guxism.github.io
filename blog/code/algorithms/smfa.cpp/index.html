符号定义

  1. |w| 表示 w 的长度
  2. w⊐x 表示 w 是 x 的后缀 suffix (CLRS e3 P.986)

  3. 状态机 M 是一个 5-tuple (Q,q0,A,Σ,δ)

    • Q: 状态集合
    • q0: 初始状态
    • A⊆Q: Accepting states 集合
    • Σ: 有限的字符集
    • δ: 状态转换函数

  4. ϕ: finite-state function, 计算状态, 是 δ 的封装;

    • ϕ(ε)=q0
    • ϕ(wa)=δ(ϕ(w),a), wa 表示字符串 w 和字符 a 的拼接
  5. σ(x)=max{k:Pk⊐x}, 其中, x 表示字符串, P 表示字符串匹配的模式, Pk 表示子字符串 P[1..k], Pk⊐x 表示 Pk 是 x 的后缀; 这里是自然语言失效的地方, 没有能表达清楚的方法: 返回一个数字 k, 这个数字切割 P, 得出 Pk 是 x 的后缀, k 是满足这一条件的 最大数字
  6. 定义 String-Matching Automaton:
    • (6.1) Q 表示状态集合 {0,1,…,m}, q0 表示状态 0, m 表示唯一的 Accepting state, 即成功匹配模式 P
    • (6.2) 设计 δ 使得 δ(q,a)=σ(Pqa) 成立, 其中
      • q 表示任意状态
      • a 表示任意输入, a∈Σ∗
    • (6.3) 不变量: ϕ(Ti)=σ(Ti), 其中:
      • T 表示 输入的字符串
      • Ti 表示结尾字符的索引为 i 的子字符串
      • 推导:
        • ϕ(Ti)=ϕ(Ti−1a) (拼接)
        • =δ(ϕ(Ti−1),a) (根据第 4 点)
        • =σ(Ti−1a) (根据 6.2)
        • =σ(Ti)
      • 含义: 每个状态的数值等于一种具体的计算结果 σ (看它定义)

一言概之, 以上是一种协议, 通过状态机的方式, 记录输入字符和匹配模式之间的关系, 可以理解为咬合关系

匹配算法

自己推导一遍容易理解:

(算法导论 32.7 图)

P = a b a b a c a, 当输入达到 T[1..6] = a b a b a b 的时候, 对于最后一个输入 b, 状态为 4, 因为 P[1..4] = a b a b 是 T[1..6] 的后缀

总的算法:

# T: 输入字符串
# transfunc: 状态转换函数
# m: accepting state, 即匹配模式的长度
FINITE-AUTOMATON-MATCHER(T, transfunc, m)
1    n = T.length
2    q = 0
3    for i = 1 to n
4        q = transfunc(q, T[i])  # 根据状态 q 和当前字符 a = T[i]
5        if q == m
6            print "Pattern occurs with shift" (i - m)

算法复杂度为 O(n)

匹配函数

void finite_automaton_matcher(string T, tmap_t &tmap, state_t accepting_state)
{
    for (size_t i = 1, q = 0; i <= T.size(); ++i)
    {
        input_t a = T[index(i)];
        q = tmap[q][a];
        if (q == accepting_state)
        {
            cout << "Pattern occurs with shift " << (i - accepting_state)
                 << endl;
        }
    }
}

从状态机到 KMP

回到起点, 已知状态机的转换状态表的计算是 O(m3|Σ|) (具体为 O(m×|Σ|×m2)), KMP 在状态机基础上把状态变换计算改良成 O(n);

核心思想:

对于模式 P[1..m], π 是 P 的前缀计算函数(prefix function), 使得:

π[q]=max{k:k<q and Pk⊐Pq}

即, 对已经匹配了 q 个字符的 T 而言, 如果下一个输入字符不匹配, 那么下一步应该移动让 P 的前缀刚好是匹配 T 的后缀, 然后再次对比, 如果还不匹配, 继续移动.

vector<state_t> computer_prefix_function(string pattern)
{
    size_t m = pattern.size();
    vector<state_t> pi(m);
#

k 表示匹配个数, 如果只匹配一个字符, 那么下一步 k 肯定是 0, 因为再移动就没重叠空间了, 因此 state = 1, 即索引等于 0 的时候 k = 0

    pi[index(1)] = 0;
    state_t k = 0;
    size_t q;
#

从匹配 2 个开始, 匹配 1 个情况不用算了

    for (q = 2; q < m; ++q)
    {
# 
如果重叠部分下一个字符不相等, x != y, 则找退路
[][][][][ ][x][]
    [][][ ][y][]
         ╰ k
移动至:
[][][][][ ][x][]
        [ ][?][][y][]
         ╰ k'
继续对比 k'+1 和 x

减 1 等于索引

        while (k > 0 && pattern[index(k+1)] != pattern[index(q)])
            k = pi[index(k)];
# 
k = 0 时, 如果能满足以下条件, 则说明 k = 1, 即重叠一个
      ╭ q = state, state 与索引相差 1, 所以 -1
T: [][ ][][]
P:   [ ][][][][]
      ╰ k+1 == 0

        if (pattern[index(k+1)] == pattern[index(q)])
            k = k + 1;
#

那么对 p[q] 而言它的下一步退路是 p[k]

    ╭ q
T: [ ][x][][]
P: [x][ ][][][]
    ╰ k

        pi[index(q)] = k;
    }
    return pi;
}

void kmp_match(string T, string P)
{
    size_t n = T.size();
    state_t m = P.size();

    auto pi = computer_prefix_function(P);
#

q 表示模式的匹配进度, q == m 时完全匹配; q = 0 表示没有匹配;

    state_t q = 0;
    for (size_t i = 1; i <= n; ++i)
    {
#

找退路

        while (q > 0 && P[index(q+1)] != T[index(i)])
        {
            q = pi[index(q)];
        }
        if (P[index(q+1)] == T[index(i)])
        {
            q = q + 1;
        }
        if (q == m)
        {
            cout << "Pattern occurs with shift " << (i - m) << endl;
            q = pi[index(q)];
        }
    }
}

void kmp_hook()
{
    string P = "ababaca";
    string T = "abababacaba";
    kmp_match(T, P);
}
