二叉查找树

红黑树

红黑树的属性很多资料也提到了. 我不知道它为什么这么设计, 为什么两种颜色能保持树的平衡, 我个人把它看作是一种经验/技巧/设计, 因此我其实只需记住它的属性, 关键操作就可以了

  1. 每个节点要么是红色要么是黑色
  2. 根节点是黑色
  3. 每个叶子是黑色
  4. 如果节点是红色的, 那么它的子节点必须都是黑色
  5. 对于每个节点, 所有到叶子的路径包含的黑色节点数都一样

                node = node->left;

                node = s.top();
                if (node->right)

                    node = node->right;

                else
                {
                    if (node->parent)
                    {
                        if (node == node->parent->left)
                        {
                            node->parent->left = nullptr;
                        }
                        else
                        {
                            node->parent->right = nullptr;
                        }
                    }
                    s.pop();
                    delete node;
                    node = nullptr;
                }

        }
    }

Insertion

    Node *insert_at(Node *node, T &v)
    {
        Node **side = v < node->value ? &(node->left) : &(node->right);
        if (*side)
        {
            return insert_at(*side, v);
        }
        else
        {
            Node *t = new Node(v);
            t->parent = node;
            *side = t;
            return t;
        }
    }

    template <typename Arg> TreeNode<T> *insert(Arg &&v)
    {
        if (this->root)
        {
            return this->insert_at(this->root, v);
        }
        else
        {
            this->root = new Node(v);
            return this->root;
        }
    }

    static Node *tree_minimum(Node* x)
    {
        auto curr = x;
        while (curr->left)
        {
            curr = curr->left;
        }
        return curr;

Transplant

目的: 删掉 u 树, 用 v 树取而代之

    void transplant(Node *u, Node *v)
    {
        if (!u->parent)
        {
            this->root = v;
        }
        else if (u == u->parent->left)
        {
            u->parent->left = v;
        }
        else
        {
            u->parent->right = v;
        }
        if (v->parent)
        {
            v->parent = u->parent;
        }
    }

Deletion

目的: 删掉 z 树

case 1: z 没有左子树, 直接删掉 z, 将 z.r 移上来

case 2: 与上一种是镜像关系

case 3: r 恰好没有左树

case 4: 找到符合 y.left == NIL 的节点 y, 可以看出这是最关键的条件: 找到左子树为空的节点, 让它接替被删节点

    void del(Node *z)
    {
        if (z->left == nullptr)
        {
            transplant(z, z->right);
            delete z;
        }
        else
        {
            if (z->right == nullptr)
            {
                transplant(z, z->left);
                delete z;
            }
            else
            {
                Node *y = Tree::tree_minimum(z->right);
                if (y->parent != z)
                {
                    transplant(y, y->right);
                    y->right = z->right;
                    y->right->parent = y;
                }
                transplant(z, y);
                y->left = z->left;
                y->left->parent = y;
                delete z;
            }
        }
    }

广度优先遍历

    std::vector<T> to_vector()
    {
        std::vector<T> res;
        std::queue<Node *> q;
        q.push(this->root);

        while (!q.empty())
        {
            Node *curr = q.front();
            q.pop();
            res.push_back(curr->value);
            if (curr->left)
            {
                q.push(curr->left);
            }
            if (curr->right)
            {
                q.push(curr->right);
            }
        }
        return res;
    }

首先对照红黑数的属性, 考虑新插入的节点会违反哪些属性:

  • 不会违反属性 1 (所有节点要么是红色要么是黑色)
  • 不会违反属性 3 (叶子 null 节点为黑色)
  • 不会违反属性 5 (任意节点到叶子节点的路径中黑色节点数一样), 因为新插入的节点是红色, 对于路径来说是透明的

因此只可能违反属性 2(根节点为黑色), 以及属性 4(红色节点的子节点必须都是黑色)
